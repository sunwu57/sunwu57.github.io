    hackerone 挖洞之旅

    1. 今天心血来潮，打算自动化挖一下 hackerone
    2. hackerone 是一个国外的漏洞悬赏平台地址hackerone

    思路

    4 万多 url，跑了几天 还没跑完没怎么休息过 针对国外资产

    子域名＞爆破＞ ssl 证书查询子域名＞ ip 池+梯子+使用爬虫对 ip 归属地进分流确保自己是该资产归属地的原生 ip（例如 hk 原生 ip 梯子访问 hk）一般国外厂商会屏蔽梯子不许访问一些页面＞ ip 段＞资产存活自动监控梳理资产＞收集敏感 js 文件以及接口告警＞收集了很多国外的 cms blog 的特征＞基础信息：IP，网段，域名，端口

    系统信息：操作系统版本

    应用信息：各端口的应用，例如 web 应用，邮件应用等

    版本信息：所有探测到的版本

    服务信息：服务器类型、版本

    人员信息：域名注册人员信息，web 应用中网站发帖人的 id，管理员姓名等

    （网站以前看到过复制的几个基本信息）

    ＞编写和 copy 了大量网上现存的 poc 以及 ChatGPT 帮忙编写存入自己轮子一次扫描，伪装 Google 爬虫（一般国外平台识别是用户会自动跳转到登录平台 Google 爬虫不会＞ xray+awvs 两次扫描＞生成报告＞每两个周重新梳理监测是否上线新的 ip 以及域名

    现在加上一个 fofa 采集和 shodan 采集

    这些主要是看自己的经验，所谓的自动化本质就是在原经验的基础上，重复的事情简单化，没啥特别牛的 网上大部分教程都是割韭菜的，比如刷漏洞，为什么很多人批量搞不出来，1.没有针对国内国外的主流编写对应的绕过脚本，比如安全狗 爬虫采集 url 到是安全狗之后，使用 Python 执行 sqlmap 自己编写的安全狗 bypass 脚本，如果你编写了主流的安全绕过脚本 基本随便搞

    爬一下有赏金的域名，然后 fofa shodan 采集一波资产

    再针对性写 cloudflare 的绕过 waf

    一般都能爬出来很多 xss

    sql 注入把带 ❓ 问号的 url 提取出来写个 bypass 脚本直接开搞

    sqlmap 批量

    流程

    确定目标

    首先是 projectdiscovery

    这里会有 Hackerone\bugcrowd\以及其他平台的自建安全计划等等

    https://chaos.projectdiscovery.io/#/

    勾选一下 with rewards，就是有奖励的资产，更新时间一般都是近几小时，找更新时间近的挖

    域名收集的工作 projectdiscovery 已经帮我们做好了，然后就直接下载即可，域名与子域名就全部下载回来了

    bounty-targets-data

    https://github.com/arkadiyt/bounty-targets-data

    这是个 git 项目，不用自己部署，用在线的就行

    每 30 分钟爬取一次有奖励的资产，并同步到 bounty-targets-data 项目上

    来源也比较多

    • Hackerone
    • Bugcrowd
    • Intigriti
    • Hackenproof
    • YesWeHack
    • Federacy

    A 类 比较严重直接资产：https://github.com/arkadiyt/bounty-targets-data/blob/master/data/domains.txt

    B 类 可以往子域名搞搞的：https://github.com/arkadiyt/bounty-targets-data/blob/master/data/wildcards.txt

    HackerOne 筛选

    Hackerone 也不是所有资产都有赏金，需要筛选一下，有赏金的，响应效率高的，有域名的

    https://hackerone.com/directory/programs?offers_bounties=true&high_response_efficiency=true&active_only=false&asset_type=URL&order_direction=DESC&order_field=resolved_report_count

    如果你是代码审计方向的师傅，想通过代码审计来获取赏金，那你就可以筛选能提供源代码让你审计的项目

    如果你是 二进制，逆向，客户端等方向的师傅，那你就筛选客户端，安卓，IOS，等等
